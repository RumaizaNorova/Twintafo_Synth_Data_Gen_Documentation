# Use cases

## Workflow validation (primary)
- Validate that diagnostics → gating → estimation → reporting behaves as expected under controlled scenarios.
- Generate consistent “review bundles” for integration testing and reviewer walkthroughs.

## Operating characteristics (OC) and sensitivity
- Quantify how often gating passes/fails under known scenario shifts.
- Summarize aggregate error/bias behavior and borrowing/ESS behavior where enabled.

## Education & demonstrations (public-safe)
- Demonstrate end-to-end clinical review workflows without exposing real patient data.

!!! warning
    Synthetic data should not be used for direct clinical decision-making unless explicitly validated and authorized for that use.
